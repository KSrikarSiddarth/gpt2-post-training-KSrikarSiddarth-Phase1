# GPT2-SFT Assignment

Use the provided SPEC.md file to build a full app for a post-trained GPT2 using any dataset of your choice. You need to add specifications to visualize intermediate states, and run inference on a local system. To get GPU for training, prepare all the codes locally on your system as .py files, upload to Kaggle, and use one kaggle notebook to just run the python training, download the trained checkpoint back to your system and continue. 

You must be creative, bring your taste and visualize intermediate states (there are some starter tasks in the SPEC.md file). THe minimum expectation is that you are able to use the provided SPEC file and create a standard SFT pipeline. For higher learning and to make yourself stand out, be creative and bring key insights that you got. 

NOTE: Being creative doesn't mean doing 20 things, you can just do 1 think extra, but make it interesting.

Use the same Agentic Coding session to fully develop your code. 

Deliverables (in a directory labelled Deliverables): 
- Updated SPEC.md
- /export (or export the chat in codex)
- Generated CLAUDE.md (or agents.md if you are using Codex)
- Claude /usage (or codex /status) 
- Plots –> Directory containing the visualizations that you created
- Reflection.md – Containing what you learnt in this assignment (max of 300 words)
